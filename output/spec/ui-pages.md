# UI Pages

Page/route tree for the modernized Northwind Traders Web App. All pages are Razor views served by the ASP.NET Core Web App. The Web App calls the API App server-side (never directly from the browser). Tech stack: Bootstrap CSS/JS, HTML, JavaScript.

Legacy forms and reports are mapped to modern routes below.

---

## Navigation Structure

```
/ (Home/Dashboard)
  |
  +-- /orders              Orders list
  |     +-- /orders/create       New order form
  |     +-- /orders/{id}         Order detail / edit
  |     +-- /orders/{id}/invoice Invoice print view
  |
  +-- /customers           Customer list
  |     +-- /customers/create    New customer form
  |     +-- /customers/{id}      Customer detail / edit
  |
  +-- /products            Product list
  |     +-- /products/create     New product form
  |     +-- /products/{id}       Product detail / edit
  |
  +-- /employees           Employee list
  |     +-- /employees/create    New employee form
  |     +-- /employees/{id}      Employee detail / edit
  |
  +-- /reports             Reports hub
  |     +-- /reports/sales-by-product          Sales by product (monthly)
  |     +-- /reports/sales-by-product-quarterly Sales by product (quarterly)
  |     +-- /reports/sales-by-employee         Sales by employee (monthly)
  |     +-- /reports/employee-email-list       Employee email directory
  |     +-- /reports/employee-phone-list       Employee phone directory
  |
  +-- /admin               Admin panel
        +-- /admin/settings    System settings editor
```

Navigation bar is present on all pages with links to: Orders, Customers, Products, Employees, Reports, Admin.

## Pages

### Home / Dashboard

- **Route:** `/`
- **Title:** Dashboard
- **Legacy form:** `frmMainMenu` (dashboard summary area)
- **Purpose:** Application landing page. Displays year-to-date sales summary and recent orders list.
- **API endpoints consumed:**
  - `GET /api/reports/sales-ytd` - YTD total revenue
  - `GET /api/reports/sales-by-employee` - YTD revenue per salesperson
  - `GET /api/orders?top=20` - 20 most recent orders
- **Key UI components:**
  - YTD revenue total card
  - Revenue by salesperson table
  - Recent orders table (OrderDate, Customer, Employee, Status, Total) with links to order detail
- **Navigation:** Top nav bar with links to all sections

### Orders List

- **Route:** `/orders`
- **Title:** Orders
- **Legacy form:** `sfrmMainMenu_OrderList`
- **Purpose:** Browse and filter all orders.
- **API endpoints consumed:**
  - `GET /api/orders` with optional filters
  - `GET /api/customers/lookup` - customer filter dropdown
  - `GET /api/employees/lookup` - employee filter dropdown
  - `GET /api/orderstatuses` - status filter dropdown
- **Key UI components:**
  - Filter bar: Customer (dropdown), Employee (dropdown), Status (dropdown), Order Date (date input)
  - Orders table: OrderDate, Customer, Employee, Status, Total - with links to detail
  - "New Order" button linking to `/orders/create`
- **Interactions:** Filter form submission reloads/filters the table

### Order Detail / Edit

- **Route:** `/orders/{id}`
- **Title:** Order #{id}
- **Legacy form:** `frmOrderDetails` + `sfrmOrderDetails`
- **Purpose:** View and edit an order header and its line items. Print invoice.
- **API endpoints consumed:**
  - `GET /api/orders/{id}` - order with line items
  - `GET /api/customers/lookup` - customer dropdown
  - `GET /api/employees/lookup` - employee dropdown
  - `GET /api/orderstatuses` - status dropdown
  - `GET /api/products/lookup` - product dropdown for line items
  - `PUT /api/orders/{id}` - save order header
  - `POST /api/orders/{id}/details` - add line item
  - `PUT /api/orders/{id}/details/{detailId}` - update line item
  - `DELETE /api/orders/{id}/details/{detailId}` - delete line item
  - `DELETE /api/orders/{id}` - delete order
- **Key UI components:**
  - Order header form: Customer (dropdown), Employee (dropdown), Order Date, Shipped Date, Paid Date, Status (dropdown), Notes textarea
  - Line items table: Product (dropdown), Quantity, Unit Price, Extended Price; Add row button; Delete row button per line
  - Order total summary (calculated, read-only)
  - "Print Invoice" button linking to `/orders/{id}/invoice`
  - Save / Delete buttons
- **Interactions:**
  - When product is selected in a line item, auto-fill Unit Price from product lookup
  - Extended price calculated client-side as Quantity x UnitPrice

## Legacy UI Features Not Carried Forward

| Legacy Feature                | Reason Dropped                                                                      |
| ----------------------------- | ----------------------------------------------------------------------------------- |
| frmStartup (splash)           | Web apps do not use splash screens; loading is immediate                            |
| frmLogin (Windows auth)       | No Windows identity in a web app; authentication is via managed identity (API-side) |
| frmWelcome                    | Welcome content was specific to the Access application; not needed in a web app     |
| frmAbout                      | Version info can be in a footer; a separate page is not warranted                   |
| frmCredentials                | Credential display was specific to the Windows login mechanism                      |
| frmNorthwindFeatures          | In-app help browser; web apps use standard documentation/help links                 |
| sfrmAdmin_ResetDates          | Date-reset was a demo workaround for Access template data; not needed in a web app  |
| frmHidden                     | Access-specific utility form; no equivalent concept in web                          |
| macFilterOrders filter combos | Replaced by standard query parameter filters on the Orders list page                |

## Assumptions

- There is no authentication UI (login page) in the Web App for local development mode, as specified in `copilot-instructions.md`. Users access the app directly.
- Chart rendering for sales reports uses a client-side chart library loaded from CDN. The specific library (e.g., Chart.js) is chosen by the WebDev Agent; CDN `<script>` tags must not include integrity attributes per the `copilot-instructions.md` policy.
- The Admin panel is accessible to any user in local development (no role checks). This matches the legacy behavior where `frmAdmin` had no programmatic access guard.
- All detail/edit pages use a single route (`/customers/{id}`, `/orders/{id}`, etc.) for both view and edit. There is no separate read-only view vs. edit view; the form is always editable.
- Confirmation dialogs before destructive actions (delete) are implemented client-side using the browser's built-in `confirm()` dialog or a Bootstrap modal.
- The employee list page is a new addition not present as a standalone form in the legacy app, but the data was accessible via the employee detail form navigation. The "Employees" section is required to support full CRUD operations.
