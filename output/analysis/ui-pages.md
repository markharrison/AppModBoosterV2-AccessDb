## Overview

The application contains **30 forms** (14 top-level, 16 subforms) and **6 reports**. Navigation is menu-driven: the main menu (`frmMainMenu`) acts as the application shell, hosting named subform panes that switch by changing the `SourceObject` property. Detail forms open as dialog windows.

### frmLogin

- **Purpose**: Employee authentication. Matches the Windows login name to `Employees.WindowsUserName` and stores the resolved `EmployeeID` in `TempVars!UserID`.
- **Record source**: None (uses `DLookup` against `qryEmployeeLogin`)
- **Controls**: Display-only text boxes (employee name, job title resolved after login); no editable password field.
- **Behavior**:
  - On open: reads `Environ("USERNAME")`, looks up matching employee.
  - If no match: falls back to `DEFAULT_LOGIN_ID = 2`.
  - Sets `TempVars!UserID` for the session duration.
  - On close: opens `frmMainMenu`.
- **Validation**: Presence of matching `WindowsUserName`; no password challenge.

## User Workflows

### Placing a New Order

1. Open `frmOrderDetails` via "New Order" on `sfrmMainMenu_OrderList`.
2. Select Customer from `qrycboCustomers` combo.
3. Select Employee (salesperson) from `qrycboEmployees` combo.
4. Set OrderDate (defaults to current date).
5. Set StatusID to "New" (`osNew = 3`).
6. Add line items in `sfrmOrderDetails`: select Product, enter Quantity; UnitPrice auto-fills from `qrycboProducts`.
7. Save - data macro records AddedBy/AddedOn on Order and OrderDetails.

### Processing an Order

1. Open existing order in `frmOrderDetails`.
2. Change StatusID through the workflow states: New -> Invoiced -> Shipped -> Paid -> Closed.
3. Set ShippedDate and PaidDate when transitioning to those states.
4. Print invoice via "Print" button (opens `rptInvoice` filtered by OrderID).

### Managing Customers

1. Browse via `frmCustomerList` (search by name/city/state).
2. Open detail in `frmCustomerDetail`.
3. Add new customer, edit contact details, review order and product history in subforms.

### Running Reports

1. Navigate to Reports tab in `sfrmMainMenu_Reports`.
2. Optionally set date range criteria.
3. Click report button to open the chosen report in preview mode.

### Admin Operations

1. Open `frmAdmin` (via hidden navigation or direct DoCmd.OpenForm call).
2. Create internet orders (random test data).
3. Reset order dates to current period.
4. Edit system settings (tax rate, welcome screen toggle).
5. Delete all test data to reset the demo database.
