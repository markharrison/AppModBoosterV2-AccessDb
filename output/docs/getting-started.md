## Local Setup

### 1. Clone the Repository

```powershell
git clone <repository-url>
cd <repository-name>
```

### 2. Restore Dependencies

```powershell
# Restore API App
dotnet restore output/src/ApiApp/ApiApp.csproj

# Restore Web App
dotnet restore output/src/WebApp/WebApp.csproj
```

### 3. Start the API App

The API App must be started **first** because the Web App depends on it.

```powershell
cd output/src/ApiApp
dotnet run --environment local
```

The API App will:
- Start on `http://localhost:5201`
- Create a SQLite database file (`northwind-local.db`) automatically
- Seed sample data (9 employees, 6 customers, 10 products, 12 orders) on first startup
- Enable Swagger UI at `http://localhost:5201/swagger`
- Expose a health check at `http://localhost:5201/health`

Verify it's running:

```powershell
# In a new terminal
Invoke-WebRequest -Uri http://localhost:5201/health -UseBasicParsing
```

### 4. Start the Web App

In a **separate terminal**:

```powershell
cd output/src/WebApp
dotnet run --environment local
```

The Web App will:
- Start on `http://localhost:5200`
- Connect to the API App at `http://localhost:5201` (configured in `appsettings.local.json` via `ApiApp:BaseUrl`)

Open your browser to **http://localhost:5200** to use the application.

### Key Configuration Settings

| Setting | Location | Purpose |
| --- | --- | --- |
| `ApiApp:BaseUrl` | Web App appsettings | URL the Web App uses to call the API App |
| `DatabaseProvider` | API App appsettings | `Sqlite` (local) or `SqlServer` (cloud) |
| `ConnectionStrings:DefaultConnection` | API App appsettings | Database connection string |
| `ASPNETCORE_ENVIRONMENT` | Environment variable | Controls which appsettings file is loaded |

### Unit Tests (87 tests)

```powershell
cd output/test/unit
dotnet test --verbosity normal
```

The unit tests use NUnit with an in-process `WebApplicationFactory` backed by a temporary SQLite database. They cover all 8 controllers, CRUD operations, edge cases, business rules (unit price snapshot, cascade delete, default status), and the data seeder.

### E2E Tests (25 tests)

E2E tests require both apps to be running locally first.

**Step 1: Start both apps** (in separate terminals):

```powershell
# Terminal 1 — API App
cd output/src/ApiApp
dotnet run --environment local

# Terminal 2 — Web App
cd output/src/WebApp

```

**Step 2: Install Playwright browsers** (first time only):

```powershell

dotnet build
pwsh bin/Debug/net10.0/playwright.ps1 install chromium
```

**Step 3: Run the tests:**

```powershell
cd output/test/e2e
dotnet test --verbosity normal
```

The E2E tests use Playwright with Chromium and cover:
- **Page load tests** (17): All 20 application pages load successfully
- **Navigation tests** (2): Navbar links work correctly
- **CRUD flow tests** (6): Create, read, update, delete operations and filtering

## Troubleshooting

### Web App shows connection errors

**Cause**: The API App is not running or not reachable at the configured URL.

**Fix**: Ensure the API App is started first on port 5201:
```powershell
cd output/src/ApiApp
dotnet run --environment local
```

Verify: `Invoke-WebRequest -Uri http://localhost:5201/health -UseBasicParsing`

### Port already in use

**Cause**: Another process is using port 5200 or 5201.

**Fix**: Find and stop the process:
```powershell
Get-NetTCPConnection -LocalPort 5200 | Select-Object OwningProcess
Get-NetTCPConnection -LocalPort 5201 | Select-Object OwningProcess
Stop-Process -Id <PID>
```

### Database errors on startup

**Cause**: SQLite database file is corrupted or migrations are out of date.

**Fix**: Delete the SQLite database file and restart the API App:
```powershell
Remove-Item output/src/ApiApp/northwind-local.db -ErrorAction SilentlyContinue
cd output/src/ApiApp
dotnet run --environment local
```

The database will be recreated and seeded automatically.

### E2E tests fail to connect

**Cause**: Both apps must be running before E2E tests start. The test base URL is hardcoded to `http://localhost:5200`.

**Fix**: Ensure both apps are running on the correct ports, then re-run the tests.

### Playwright browser not found

**Cause**: Chromium browser has not been installed for Playwright.

**Fix**:
```powershell
cd output/test/e2e
dotnet build
pwsh bin/Debug/net10.0/playwright.ps1 install chromium
```

### Wrong environment loaded

**Cause**: `ASPNETCORE_ENVIRONMENT` is not set or set incorrectly.

**Fix**: Use `--environment local` when running locally:
```powershell
dotnet run --environment local
```

Or set the environment variable:
```powershell
$env:ASPNETCORE_ENVIRONMENT = "local"
dotnet run
```
