# Getting Started - Northwind Traders

This guide walks you through running the Northwind Traders application on your local machine.

## Clone and Set Up

```bash
git clone https://github.com/your-org/your-repo.git
cd your-repo
```

No additional setup is required for local development. The API App uses a local SQLite database (`northwind-local.sqlite`) which is seeded automatically on first startup.

## Running Locally

### Option 1: Use the run-local Script (Recommended)

The `run-local.ps1` script builds the solution and starts both apps as background jobs, streaming their combined output to your terminal.

```powershell
.\output\scripts\run-local.ps1
```

To skip the build step if you have already built:

```powershell
.\output\scripts\run-local.ps1 -SkipBuild
```

Press **Ctrl+C** to stop both apps. The script cleans up background jobs on exit.

### Option 2: Manual Start (Two Terminals)

Start the API App in the first terminal:

```bash
dotnet run --project ./output/src/ApiApp --urls http://localhost:5201
```

Start the Web App in a second terminal:

```bash
dotnet run --project ./output/src/WebApp --urls http://localhost:5200
```

Both apps must be running at the same time for the Web App to function.

## Configuration Overview

### API App (`output/src/ApiApp/appsettings*.json`)

| Key                                   | Description                               | Local Default                                                              |
| ------------------------------------- | ----------------------------------------- | -------------------------------------------------------------------------- |
| `ConnectionStrings:DefaultConnection` | Database connection string                | `Data Source=northwind-local.sqlite` (from `appsettings.Development.json`) |
| `Swagger:Enabled`                     | Enable Swagger UI                         | `true`                                                                     |
| `ASPNETCORE_ENVIRONMENT`              | Controls which appsettings file is loaded | `Development`                                                              |

### Web App (`output/src/WebApp/appsettings*.json`)

| Key                      | Description                               | Local Default           |
| ------------------------ | ----------------------------------------- | ----------------------- |
| `ApiSettings:BaseUrl`    | Base URL of the API App                   | `http://localhost:5201` |
| `ASPNETCORE_ENVIRONMENT` | Controls which appsettings file is loaded | `Development`           |

To override the API URL without editing files, set the environment variable before running:

```bash
# PowerShell
$env:ApiSettings__BaseUrl = "http://localhost:5201"

# Bash
export ApiSettings__BaseUrl=http://localhost:5201
```

Note: ASP.NET Core uses `__` (double underscore) as the hierarchy separator for environment variables.

## Running Tests

### Unit Tests

Unit tests target the API App only. They use an in-process SQLite database (no external services required).

```bash
dotnet test ./output/test/unit/ApiApp.Tests.csproj
```

Expected result: **72 tests passed**.

To run with detailed output:

```bash
dotnet test ./output/test/unit/ApiApp.Tests.csproj --verbosity normal
```

### End-to-End (E2E) Tests

E2E tests use Playwright and require both apps to be running before executing.

**Step 1**: Start both apps (see [Running Locally](#running-locally) above).

**Step 2**: In a third terminal, run the E2E test project:

```bash
dotnet test ./output/test/e2e/E2ETests/
```

Expected result: **24 tests passed**.

#### Playwright Browser Setup

If Playwright browsers are not installed, the tests will fail with a browser-not-found error. Install them with:

```powershell
# Navigate to the E2E test project directory
cd ./output/test/e2e/E2ETests

# Install Playwright browsers (Chromium only)
pwsh playwright.ps1 install chromium
```

Or install all browsers:

```powershell
pwsh playwright.ps1 install
```

## Troubleshooting

### Port Conflict: "Address already in use"

Another process is using port 5200 or 5201. Find and stop it:

```powershell
# Find what is using port 5201
netstat -ano | findstr :5201

# Kill the process (replace PID)
taskkill /PID <PID> /F
```

Then restart the apps.

### SQLite Database Not Found

If the API App reports it cannot find `northwind-local.sqlite`, ensure you are running `dotnet run` from the project directory, or that the working directory is `output/src/ApiApp/`.

The file is created automatically by EF Core on first run. If it is corrupted, delete it and restart:

```bash
del output\src\ApiApp\northwind-local.sqlite
```

The database will be recreated and seeded with sample data on next startup.

### Playwright Browser Not Found

Run the browser install command from the E2E test project directory:

```powershell
cd ./output/test/e2e/E2ETests
pwsh playwright.ps1 install chromium
```

### E2E Tests Fail with Connection Refused

Both the API App and Web App must be running before starting E2E tests. Verify:

```bash
# Check API App is responding
curl http://localhost:5201/health

# Check Web App is responding
curl http://localhost:5200
```

If either returns an error, start the missing app before rerunning tests.

### Web App Shows Errors or Empty Data

The Web App calls the API App at `http://localhost:5201` (default configuration). If the API App is not running, all pages that fetch data will show errors. Start the API App first, then reload the Web App.

### Build Errors After Pulling Changes

Restore NuGet packages and rebuild:

```bash
dotnet restore AppModBoosterV2x.sln
dotnet build AppModBoosterV2x.sln
```

### appsettings Hierarchy

Local config loads in this order (later files override earlier ones):

1. `appsettings.json` (base defaults)
2. `appsettings.Development.json` (local overrides, includes SQLite connection string)
3. Environment variables

For Azure environments, `appsettings.Development.json` is not loaded. Instead `appsettings.dev.json`, `appsettings.staging.json`, or `appsettings.prod.json` is loaded based on the `ASPNETCORE_ENVIRONMENT` app setting configured in the App Service.
