## System Overview

Northwind Traders Starter Edition v2.4 is a modernized .NET 10 application that replaces a legacy Access database with a two-tier web architecture backed by SQL Server. The system provides order management, customer tracking, product catalog, employee directory, and reporting capabilities for the Northwind Traders business.

### Component Relationships

- **Browser → Web App**: Users interact exclusively with the Web App via a browser. The Web App serves Razor views with Bootstrap 5.3 styling.
- **Web App → API App**: The Web App calls the API App server-side using `HttpClient` (configured via `ApiApp:BaseUrl`). The browser never communicates directly with the API App.
- **API App → Database**: The API App connects to the database using Entity Framework Core. In local development it uses SQLite; in cloud environments it uses Azure SQL Server with managed identity authentication.

CORS is not required because the API App is only called server-side by the Web App.

## Authentication and Authorization

### Local Development

- No authentication between Web App and API App
- SQLite database — no SQL authentication required

### Cloud Deployment

All service-to-service communication uses **managed identity (Microsoft Entra ID)** — no plaintext credentials or secrets.

```
┌──────────┐   Managed Identity   ┌──────────┐   Managed Identity   ┌───────────┐
│ Web App  │ ──────────────────> │ API App  │ ──────────────────> │ SQL Server│
│ (MI)     │                      │ (MI)     │                      │ (AD-only) │
└──────────┘                      └──────────┘                      └───────────┘
```

- **Web App → API App**: Authenticated via managed identity
- **API App → SQL Server**: Connection string uses `Authentication=Active Directory Managed Identity`
- **SQL Server**: Azure AD-only authentication enabled — no SQL auth (`azureADOnlyAuthentication: true`)
- **CI/CD**: GitHub Actions authenticate to Azure using OIDC federated credentials (no stored secrets)

### MCAPS Policy Compliance

- Azure AD-only authentication on SQL Server — no `administratorLogin`/`administratorLoginPassword`
- Managed identities for all service-to-service auth — no plaintext credentials
- TLS 1.2 minimum enforced on all web apps and SQL Server
- HTTPS-only on all web apps

## Infrastructure Summary

All Azure resources follow the naming convention `{resourcetype}-{app}-{env}`.

## Data Model Overview

The API App manages 7 entities:

| Entity | Description |
| --- | --- |
| **Customer** | Customer records with contact information |
| **Employee** | Employee directory |
| **Order** | Customer orders with date, employee, and status tracking |
| **OrderDetail** | Line items within an order (product, quantity, unit price) |
| **Product** | Product catalog with pricing and stock |
| **OrderStatus** | Lookup table with 5 statuses (seeded via `HasData`) |
| **SystemSetting** | Application settings (e.g., TaxRate) |

Key relationships:
- Order → Customer (many-to-one)
- Order → Employee (many-to-one)
- Order → OrderStatus (many-to-one)
- OrderDetail → Order (many-to-one, cascade delete)
- OrderDetail → Product (many-to-one)

Database seeding with sample data runs automatically on startup in `local` and `dev` environments (9 employees, 6 customers, 10 products, 12 sample orders).
